# YJAutoPostOnX 🚀

**استوديو ذكاء اصطناعي + ناشر تلقائي على X.com** — يولّد تغريدات عربية احترافية بالذكاء الاصطناعي، ويجمعها في طابور مشترك، ثم ينشرها تلقائياً عبر عدة حسابات X — كل ذلك من واجهة عربية RTL واحدة.

`Electron 30` · `Playwright` · `v5.13.0` · `MIT`

---

## 🧭 كيف يعمل النظام

ثلاث مراحل مستقلة تماماً عن بعضها:

| المرحلة | الدور |
|---|---|
| 🧠 **استوديو الذكاء** | يولّد التغريدات بالذكاء الاصطناعي، يفلترها ببوابة جودة صارمة، يمنع تكرار المعنى نهائياً، **ويحقن رابط الإحالة في النص وقت التوليد** |
| 📋 **الطابور المشترك** | يستقبل المقبول من الاستوديو (أو استيراد CSV) — مراجعة، حذف، تصدير — طابور استهلاكي: النشر يبدأ دائماً من أول منشور، وكل منشور يُنشر بنجاح يُحذف من الطابور فوراً ونهائياً |
| 🚀 **محرك النشر** | يأخذ المنشورات من الطابور وينشرها على X **كما هي حرفياً** — مستقل 100% عن الإحالة: لا بوابة ولا تعديل روابط ولا رفض منشورات |

## ✨ المميزات

### 🧠 استوديو الذكاء الاصطناعي

- **توليد بالذكاء الاصطناعي فقط** — تغريدات عربية بطول 170–270 حرفاً، مع إيموجي وهاشتاقات ورابط إحالتك مضمّناً تلقائياً
- **جلسات متوازية بلا حد أقصى** — العدد الذي تختاره يعمل فعلاً (50، 500، أكثر…) وقابل للرفع والخفض **أثناء التوليد**؛ الإطلاق متدرّج والواجهة تبقى سلسة مع مئات الجلسات
- **منع تكرار المعنى نهائياً** — فهرس دلالي بترجيح IDF وتطبيع عربي كامل (تشكيل، همزات، جذور) يقارن كل تغريدة جديدة ضد كل ما نُشر سابقاً + الطابور والمعاينة وكل ما وُلّد — بلا نافذة زمنية، ويرفض التطابق الحرفي وإعادة الصياغة ونفس الفكرة بكلمات أخرى، بسرعة ثابتة حتى مع 100 ألف+ تغريدة
- **بوابة جودة صارمة** — رفض تلقائي للنصوص القصيرة/الطويلة، الإيموجي المكسور، النص الإنجليزي المتسرب من تفكير الموديل، المسودات والأفكار غير المكتملة
- **برومبت مخصص اختياري** + معاينة كاملة قبل الإضافة للطابور، مع إيقاف يحتفظ بالمقبول

### 🚀 محرك النشر

- **مستقل تماماً عن الإحالة** — وظيفته الوحيدة: سحب من الطابور والنشر كما هو؛ إعدادات الإحالة لا تؤثر على النشر إطلاقاً
- **بروفايلات مرقّمة إجبارياً** — الافتراضي رقم 1 دائماً، وكل جديد يأخذ الرقم التالي (`2- محمد`، `3- علي`…)؛ التشغيل يبدأ من البروفايل المختار ويكمل لمن **بعده** فقط بالترتيب — لا يلتف أبداً للخلف
- **معالجة ذكية لحد النشر (Rate Limit)**:
  - كشف رابط الحد اليومي (`daily_post_limit`) وكشف نصّي فوري من أول محاولة فاشلة
  - **تسليم المنشور** الذي ضرب الحد للحساب التالي فوراً بدل فقدانه — مع تقدّم مؤشر الحساب الأصلي حتى لا يتكرر
  - كولداون محفوظ لكل بروفايل (يُستخرج من رسالة X نفسها عند توفرها) مع عدّاد حي وإلغاء يدوي
  - "قاطع دائرة" يوقف الحساب احترازياً بعد 3 فشلات متتالية غير مبررة
- **إيقاف فوري حقيقي** — زر الإيقاف يقاطع كل الانتظارات والمحاولات خلال ≤200ms
- **سلوك مضاد للكشف** — تأخيرات عشوائية، حركة فأرة طبيعية، تمرير بشري، و3 محاولات لكل منشور

### 📋 الطابور والتقارير

- طابور JSON مشترك محمي بقفل، مع قوائم افتراضية (virtual lists) تتعامل بسلاسة مع آلاف المنشورات
- **الطابور يُستهلك**: كل منشور يُنشر مرة واحدة فقط — بمجرد نجاح النشر والتقاط الرابط يُحذف من `queue.json` نهائياً ولا يعود إطلاقاً. المنشور الذي لم يُنشر يبقى في مكانه، فيلتقطه الحساب التالي أو التشغيل التالي تلقائياً
- استيراد/تصدير CSV، حذف جماعي، عدّاد حروف لكل منشور، ودعم Spintax‏ `{خيار1|خيار2}` في النصوص
- تقارير تشغيل حية: نسبة مئوية + عداد كسري + سجلات CSV لكل جلسة

### 🔒 الأمان

- تدقيق أمني تلقائي (`npm run audit`) — فحص روابط مثبّتة بالكود وسلامة ملفات الحالة
- مفاتيح API تُحفظ محلياً فقط ولا تُسرَّب في السجلات أبداً
- اكتشاف تلقائي لنوع المزوّد (OpenAI / Anthropic / Gemini) من الـ Base URL، مع مزودين جاهزين في القائمة وجلب الموديلات المتاحة بضغطة زر

## 🚀 التشغيل

> **المتطلبات:** Node.js v18+ — يعمل على Linux وmacOS وWindows

<div dir="ltr">

```bash
git clone https://github.com/yjlvfe/YJAutoPostOnX.git
cd YJAutoPostOnX
npm install
npm start
```

</div>

| الأمر | الوظيفة |
|---|---|
| `npm start` | تشغيل التطبيق |
| `npm test` | سلسلة الاختبارات الكاملة (11 ملفاً) |
| `npm run audit` | التدقيق الأمني |
| `npm run build:linux` | بناء AppImage → `dist/YJAutoPostOnX-*.AppImage` |

## 🏗️ هيكل المشروع

<div dir="ltr">

```
dev/
├── main.js              # Electron main + IPC + منسّق التوليد
├── preload.js           # جسر الواجهة (window.api)
├── automation/          # المحرك: توليد، طابور، جلسات، نشر، كولداون، إحالة، تقارير
│   ├── contentEngine.js #   بناء البرومبت + بوابة الجودة
│   ├── semanticIndex.js #   منع تكرار المعنى (IDF + فهرس معكوس)
│   ├── sessionManager.js#   الجلسات المتوازية
│   ├── queueManager.js  #   الطابور المشترك الاستهلاكي (إضافة/حذف/استهلاك)
│   ├── xPoster.js       #   النشر عبر Playwright (مستقل عن الإحالة)
│   └── …
├── security/            # التدقيق، فحص الروابط، ترحيل الإعدادات
├── ui/                  # الواجهة العربية RTL (4 شاشات)
└── config/              # قائمة مزودي AI
test/                     # 20+ ملف اختبار (بدون فريمورك)
```

</div>

**البيانات والإعدادات** تُحفظ محلياً في <code dir="ltr">~/.config/x-poster-bot-profile</code> (الإعدادات والكولداون) و<code dir="ltr">~/.config/x-poster-shared</code> (الطابور المشترك <code dir="ltr">queue.json</code> + أرشيف المنشور <code dir="ltr">published.json</code> الذي يغذّي منع التكرار).
